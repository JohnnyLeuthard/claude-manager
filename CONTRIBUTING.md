# Contributing to Claude Manager

Thank you for your interest in improving Claude Manager! This document explains how to contribute.

## How to Contribute

### Report Issues or Suggest Improvements

**Have a question or found an error?** Open an issue on GitHub.

Before opening an issue, check:
- The [README.md](README.md) might already answer your question
- Existing issues might already cover what you want to report

**What to include in an issue:**
- What you found (error, unclear section, missing information)
- Where you found it (which folder/section in README.md)
- What you expected vs. what you saw
- Your Claude Code setup if relevant (OS, VS Code version, etc.)

### Improve Documentation

**The README.md is the heart of this project.** If you use Claude Code and notice we're missing something in the `~/.claude` reference, we want to know!

**Ways you can help:**
- Report new or updated `~/.claude` folders we haven't documented
- Suggest clearer explanations for confusing sections
- Add examples or clarifications based on your experience
- Fix typos or improve readability

### Contribute Code (Phase 2+)

Once Phase 2 (cleanup scripts) starts, we'll be accepting PRs for:
- `scan.js` — improvements to folder classification
- `clean.js` — safer cleanup logic
- `audit.js` — better security checks
- Tests and documentation

For code contributions, please:
1. Fork the repo
2. Create a branch for your changes
3. Write clear commit messages
4. Include tests if applicable
5. Open a PR with a description of your changes

## Code of Conduct

Be respectful. We're all here to make Claude Code easier to use.

## Questions?

Feel free to open an issue with the `question` label.
