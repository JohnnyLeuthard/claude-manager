## How This Project is Organized

**Phase 1 (Complete):** Comprehensive reference documentation
- README.md — complete map of `~/.claude`
- Safety levels and security notes for everything
- Cleanup guidance and best practices

**Phase 2 (Planned):** Cleanup scripts
- `scan.js` — analyze your `~/.claude` folder
- `clean.js` — safe cleanup with dry-run mode
- `audit.js` — security risk detection

**Phase 3 (Planned):** Visual interface
- Interactive browser/VSCode UI
- Visual representation of what's in your folder
- Point-and-click cleanup

## Core Principles

- **Safety first** — dry-run everything before making changes
- **Privacy-aware** — documents what's sensitive and why
- **Comprehensive** — every file in `~/.claude` is documented
- **No dependencies** — works standalone, needs no special setup
