How The Workflow Works

The current tool is deliberately simple: read from disk, classify from trusted local metadata, then present a clear action surface.

1

Scan Live Disk Usage

`scan.js` reads the real folders in `~/.claude` and calculates actual sizes every run.

2

Classify Safety

Known metadata explains purpose, importance, freeable space, deletion tradeoffs, and related docs.

3

Generate A Dashboard

The HTML report turns folders into scan-friendly cards with badges, links, and cleanup context.

4

Act With Guardrails

Planned cleanup tooling is dry-run by default, with explicit execution only when the user chooses it.

Why It Is Valuable

The value is not just saving space. It is removing uncertainty from a folder that stores working memory, settings, and privacy-sensitive traces.

V

Visibility

Shows where disk usage lives and why each folder exists, so cleanup starts with understanding.

S

Safety

Separates critical data from stale or regenerated data, reducing the chance of deleting something important.

P

Privacy

Calls out transcripts, shell snapshots, backups, and other places where sensitive context can accumulate.

A

Actionability

Connects every folder to a practical recommendation: keep, review, prune old entries, or safely delete.
